## Работа со скриптом
### Что делает скрипт

Скрипт считает православные праздники и посты для заданного диапазона лет и формирует **два отдельных календаря в формате `.ics`**:

- `orthodox_feasts.ics` — **праздники**
- `orthodox_fasts.ics` — **посты** (продолжительные, однодневные, обычные ср/пт)

Файлы можно по отдельности импортировать в разные календари Google.

## Как импортировать календари в Google Calendar

Далее всё делается в веб-интерфейсе Google Calendar.

### Импорт праздников (`orthodox_feasts.ics`)

- **Открой** Google Calendar в браузере.
- Слева, рядом с блоком **«Другие календари»**, нажми на `+`.
- Выбери **«Импорт»**.
- В поле **«Выберите файл на компьютере»** укажи `orthodox_feasts.ics`.
- В поле **«Добавить в календарь»** лучше создать новый календарь (например, «Православные праздники»), если ещё не создан.
- Нажми **«Импорт»**.

### Импорт постов (`orthodox_fasts.ics`)

- Аналогично, через **«Другие календари» → + → Импорт»**.
- Выбери файл `orthodox_fasts.ics`.
- В качестве календаря выбери отдельный (например, «Православные посты»).
- Нажми **«Импорт»**.

Так у тебя будут **два независимых слоя**: праздники и посты, которые можно по отдельности включать/выключать и настраивать цвет.

## Структура и логика скрипта

### Расчёт Пасхи

- `orthodox_easter(year)` — считает дату православной Пасхи:
  - Если установлен `python-dateutil`, используется `dateutil.easter` с `EASTER_ORTHODOX`.
  - Если нет — резервный расчёт по юлианской формуле (корректен для 1900–2099).

От Пасхи считаются:
- Вербное воскресенье, Лазарева суббота, Страстная пятница
- Великий пост (начало)
- Вознесение, Троица
- Апостольский пост и сплошные седмицы после Пасхи и Троицы.

### Формирование событий `.ics`

- `_escape_ical_text(text)` — экранирует текст по правилам iCalendar (слеши, `;`, `,`, переносы строк).  
- `make_ics_event(uid, start, end, summary, description)`:
  - Создаёт один `VEVENT`:
    - `DTSTART;VALUE=DATE` — начало (включительно)
    - `DTEND;VALUE=DATE` — **день после конца** (особенность формата для целодневных событий)
  - `SUMMARY` — заголовок события.
  - `DESCRIPTION` — описание (включая ссылки и `\n` для переноса строк).

- `_emit_spans(...)` — объединяет **несколько подряд идущих дней с одинаковым типом поста** в одно событие (например, одна запись для «С маслом - Неделя N-я…» вместо 2–3 отдельных дней).

- `write_ics_file(events, filename)`:
  - Удаляет старый файл, если есть.
  - Пишет шапку `VCALENDAR`, все события `VEVENT` и закрывающий `END:VCALENDAR`.

## Основные данные и правила внутри `main()`

### 1. Справочник степеней строгости поста

```python
FAST_TYPES = {
    "post": ("Пост", "Постный день (без мясного, молочки и яиц)\nhttps://azbyka.ru/recept/\nhttps://azbyka.ru/days/p-posty-v-sredu-i-pjatnitsu"),
    "fish": ("Рыба", "Без мясного, молочки и яиц\nhttps://azbyka.ru/recept/"),
    "fish_roe": ("Рыбная икра", "Горячая пища с растительным маслом + рыбная икра\nhttps://azbyka.ru/recept/"),
    "no_oil": ("Без масла", "Горячая пища без масла\nhttps://azbyka.ru/recept/"),
    "with_oil": ("С маслом", "Горячая пища с растительным маслом\nhttps://azbyka.ru/recept/"),
    "dry": ("Сухоядение", "Холодная пища (вода, хлеб, фрукты, овощи, компоты)\nhttps://azbyka.ru/recept/osnovy-suhojadenija/\nhttps://azbyka.ru/suxoyadenie"),
  }
```

- **Названия типов поста и базовые описания** собраны в одном месте.
- Большинство событий поста используют именно этот словарь (включая ссылки на рецепты и статьи).
- **Строгие посты** (Крещенский сочельник, Усекновение, Воздвижение, Страстная пятница, строгие дни 1-й недели ВП) задаются отдельными текстами вне этого словаря.

### 2. Неподвижные праздники

Список `fixed_feasts`:

- Каждая запись: `(название, месяц, день, базовое_описание)`.
- В цикле по годам для каждого праздника:
  - Строится дата.
  - К описанию добавляется ссылка на `azbyka.ru/days/YYYY-MM-DD`.
  - Событие пишется в `feast_events`.

### 3. Сплошные седмицы

Множество `continuous_weeks`:

- Святки: 7–18 января.
- Седмица мытаря и фарисея.
- Сырная (масленичная).
- Светлая (первый недельный круг после Пасхи).
- Троицкая (после Троицы).

**В эти дни не создаются среда/пятница-посты** (но продолжающиеся посты учитываются отдельно).

### 4. Продолжительные посты

Множество `prolonged_fast_dates` собирает все даты, которые входят в:

- **Великий пост**: от понедельника за 48 дней до Пасхи по день перед Пасхой.
- **Апостольский пост**: с **понедельника после недели** после Троицы по 11 июля включительно.
- **Успенский пост**: 14–27 августа.
- **Рождественский пост**:
  - Хвост прошлого года: 1–6 января.
  - Начало текущего: 28 ноября – 31 декабря.

**Обычные ср/пт-посты не создаются для этих дат**, чтобы не дублировать события.

#### Великий пост

- Считается от `lent_monday = pascha - 48 дней`.
- Для каждой недели (1–7) собирается список правил `gl_rules`:
  - Особые дни:
    - Страстная пятница — строгий пост с отдельным описанием и ссылкой.
    - Благовещение (7 апреля, если попадает внутрь поста) — `fish_oil`.
    - Вербное воскресенье — `fish_oil`.
    - Лазарева суббота — `fish_roe`.
  - 1-я неделя:
    - Пн–Вт–Чт — строгий пост, отдельное описание.
  - Остальные:
    - Пн, Вт, Чт — `no_oil` (горячая пища без масла).
    - Ср, Пт — `dry` (сухоядение).
    - Сб, Вс — `with_oil` (с маслом).
- После заполнения `gl_rules` вызывается `_emit_spans`, чтобы **объединить** подходящие дни в интервалы.

#### Апостольский пост

- Начало: `apostolic_start = trinity + 8 дней` (понедельник после троицкой седмицы).
- Конец: `11 июля`.
- Правила:
  - Ср, Пт — `with_oil`.
  - Остальные — `fish`.
- Всё также проходит через `_emit_spans`.

#### Успенский пост (14–27 августа)

- Пн, Вт, Чт — `no_oil`.
- Ср, Пт — `dry`.
- Сб, Вс — `with_oil`.

#### Рождественский пост

Разбит на три периода с разными правилами, как ты задавал:

- **28.11–19.12**:
  - Ср, Пт — `with_oil`.
  - Остальные — `fish`.
- **20.12–01.01**:
  - Ср, Пт — `no_oil`.
  - Пн, Вт, Чт — `with_oil`.
  - Сб, Вс — `fish`.
- **2–6.01**:
  - Пн, Вт, Чт — `no_oil`.
  - Ср, Пт — `dry`.
  - Сб, Вс — `with_oil`.

### 5. Однодневные посты

Список `one_day_fasts`:

- Крещенский сочельник — 18 января.
- Усекновение — 11 сентября.
- Воздвижение — 27 сентября.

Для каждого:
- Строится дата `d`.
- Описание + `\nhttps://azbyka.ru/days/YYYY-MM-DD`.
- Событие добавляется в `fast_events` **одним днём**.

---

### 6. Обычные посты в среду и пятницу

Цикл по всем дням года:

- Если день — среда или пятница.
- И он:
  - не в `continuous_weeks`,
  - не в `prolonged_fast_dates`,
  - не совпадает с одним из однодневных строгих постов.
- Тогда создаётся событие:
  - Название: `Пост - Среда` или `Пост - Пятница`.
  - Описание берётся из `FAST_TYPES["post"][1]` (включая ссылки).

---

### 7. Праздники, зависящие от Пасхи

В `feast_events` записываются:

- Вербное воскресенье (за неделю до Пасхи) — название и описание с ссылкой.
- Пасха — «Пасха (год)», описание + ссылка.
- Вознесение — 40-й день (литургически, `+39` от Пасхи).
- День Святой Троицы — 50-й день (`+49` от Пасхи).
- Плюс все **неподвижные** праздники (`fixed_feasts`).
